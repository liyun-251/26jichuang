## 一、测试架构设计

### 1.1 单文件集成 14 项测试参数

- 将原本 12 个独立 `TP3057()` 函数整合为一个文件，覆盖 DC 参数、增益、失真三大类
- 测试流程按信号依赖关系排序：CON-TEST → FUNC-TEST → IIL → IIH → IOZ → ICC0IBB0 → ICC1IBB1 → GXA → GXR → GRA → GRR → GRRL → SFDX → SFDR → IMD

### 1.2 `{ }` 块作用域隔离（★ 本人提出）

```c
#if TEST_GXA_ENABLE
    {   /* GXA 变量仅在此作用域可见 */
        int num_byte;
        double gain_db;
        ...
    }
#endif
```

- 每个测试项用独立的 `{ }` 块包裹，变量互不冲突
- 消除跨测试的变量名污染，避免误用上一测试项的残留值

## 三、测试开关与调试控制

### 3.1 `#if` 测试项使能开关（★ 本人提出）

```c
#define TEST_CON_ENABLE       1
#define TEST_FUNC_ENABLE      1
#define TEST_IIL_ENABLE       1
...
#define TEST_IMD_ENABLE       1
```

- 将不想跑的测试项对应宏改为 `0` 即可跳过
- **应用场景**：调试单测时只开一项；批量生产时全部开启；开发中关闭耗时的 IMD 加速迭代

### 3.2 分箱使能开关（★ 本人提出）

```c
#if BIN_ENABLE
  #define DO_BIN(n) BIN(n)
#else
  #define DO_BIN(n) ((void)0)
#endif
```

- `BIN_ENABLE = 0` 时，所有 `DO_BIN()` 编译为空操作
- **应用场景**：调试时关闭分箱，芯片失败不自动踢出，方便反复测量同一颗 chip 排查问题

## 四、PCM 捕获：从静态偏移到动态帧同步

> 这是本程序**最具技术深度**的创新点之一

### 4.1 传统方案的问题（已弃用的 `legacy` 版本）

```c
capture_multi_pcm_bytes_legacy(start_idx, dx_channel, offset, ...)
// 假设第 offset 个周期就是第一个数据位 — 硬编码，不可靠
```

- 依赖图形文件中手动数出的固定 offset
- 一旦修改图形文件（改变前导周期数），offset 就失效

### 4.2 新方案：监听 FSX 动态定位（★ 本人提出）

```c
capture_multi_pcm_bytes_fsx(start_idx, dx_channel, fsx_channel, offset, ...)
```

**原理**：

1. 同时读取 **FSX 通道** 和 **DX 通道** 的 fail memory
2. 扫描 FSX 失效位：图形中 FSX 恒设为 0，当 FSX 实际输出 1 时 → 比较不匹配 → 失效位 = 1
3. 捕获到 FSX=1 后，紧跟的 8 个周期即为 DX 的 PCM 数据位（MSB-first）

**优势**：

- **自适应**：图形文件修改后无需改 offset，FSX 脉冲在哪里，数据就从哪里抓
- **鲁棒**：不依赖绝对周期号，只依赖 FSX→DX 的相对时序
- **通用**：`offset` 参数退化为"从第几个周期开始搜索"，仅用于跳过前导

### 4.3 Fail Memory 位级操作

```c
dx_level = (dx_fail_bits[dx_cycle / 8] >> (dx_cycle % 8)) & 1;
pcm_byte |= (unsigned char)(dx_level << (7 - bit));   // MSB-first 组装
```

- 直接操作 fail memory 的 bit 级数据，零拷贝提取 PCM 码流
- 支持 MSB-first（TP3057 串行格式）正确拼装

## 五、频域分析技术

### 5.1 A-law 偶数位反转解码

```c
alaw_std = tp3057_byte ^ 0x55;   // TP3057 特有：A-law + 偶数位反转
```

- TP3057 在标准 G.711 A-law 基础上额外做了偶数位 XOR 0x55
- 先还原标准 A-law，再查表/计算 G.711 分段线性解码

### 5.2 汉宁窗 + DFT 的失真分析流水线

- 统一流程：**采样 → 加窗 → DFT → bin 查找 → 失真计算**
- SFDX、SFDR、IMD 三个失真测试复用同一套函数，仅在采样来源（PCM/DVM）和参数（点数、频率）上不同

### 5.3 IMD 双对齐频率选择（可展开讲）

- 同时满足 **AS 循环整数周期** + **FFT bin 整数对齐** 的双重约束
- 推导了三套可行方案（A/B/C），当前采用 bin 密度最高的方案 B
- 四个频率分量（f₁、f₂、f_sum、f_diff）全部精确落在 bin 中心，零频谱泄漏

### 5.4 幅度谱平均降噪

- SFDR：4 次 DVM 采样 → DFT → 幅度谱累加 → 求平均
- 降低噪声方差，提高重复性（标准差降低 √N 倍）

## 六、容错与诊断设计

### 6.1 三级失败隔离

```
硬件故障（图形运行失败）  → BIN(31/32) + SHOW_RESULT + goto 终止
   ↓
算法故障（DFT/FFT 失败）  → BIN(15/19/23) + goto 终止
   ↓
指标超限（失真/增益超限） → BIN(18/22/26) + 不终止（继续后续测试）
```

- 前两级为致命错误，立即跳转到清理代码（`goto END_XXX`）
- 第三级为性能不达标，记录 BIN 后继续，不影响其他参数测试

### 6.2 每个 BIN 对应唯一故障点

```c
BIN 14 → PCM 捕获不足    BIN 15 → DFT 失败
BIN 16 → 谐波 bin 越界    BIN 17 → 基波丢失
BIN 18 → 失真超限
```

- 32 个 BIN 覆盖所有故障模式，精确到具体失败环节
- 生产环境中无需重新测量即可定位 root cause

### 6.3 CON-TEST 预防性检查

- 所有测试之前的**首项测试**：检测 VFXI+/VFRO/PWRO+/PWRO- 对地二极管压降
- 异常→立即 `return`，不跑后续 13 项 → 避免在损坏芯片上浪费时间

### 6.4 `RUN_PATTERN` 返回值检查

```c
if (!RUN_PATTERN(10, 0, 0, 0)) {
    DO_BIN(31);
    SHOW_RESULT("SFDR_PAT_FAIL", ...);
    goto END_SFDR;
}
```

- 图形运行失败即时捕获，不会用无效数据继续计算

## 七、硬件交互的精细控制

### 7.1 Loop Around 环回测量

- **IMD 测试**：利用 relay7 将 DX↔DR 短接，AS 播放双音→TX 编码→RX 解码→DVM 采样
- 全程在芯片内部完成编码-解码链路，无需外部仪器介入

### 7.2 继电器矩阵的精确管理

```c
CLOSE_RELAY("3,4,5,6");    // CON-TEST: 四路模拟脚同时测
CLOSE_RELAY("1,7");        // IMD: VFRO→DVM + DX↔DR
CLEAR_RELAY("1,7");        // 测试结束立即释放，避免影响后续测试
```

- 每次测试按需闭合/释放继电器，不残留状态

### 7.3 DVM 量程自适应

- SFDR（5.0V 量程）、IMD（2.0V 量程）、GRRL（自动切换 20mV/5V）
- 根据信号幅度选择最优量程，兼顾精度和防溢出
