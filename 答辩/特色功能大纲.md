## 一、监听 FSX 动态定位 DX PCM 输出

### 1.1 解决的问题

ST3020 ATE 运行向量图形后，DX 引脚输出的 PCM 串行数据存储在**失效存储器 (Fail Memory)** 中。要读取 PCM 字节，必须知道 DX 数据位在哪个测试周期开始输出。传统做法是打开图形文件人工数周期号，一旦修改图形（增减前导周期），offset 就失效。

### 1.2 设计思路

不依赖固定的周期偏移量，而是**利用 FSX 帧同步信号作为"路标"**：在图形文件中将 FSX 期望值设为恒 0，当 TP3057 实际输出 FSX=1 时，比较不匹配，FSX 通道的 fail memory 对应位 = 1。扫描 FSX 的 fail memory 找到这个"1"，其后的 8 个周期就是 DX 的 PCM 数据位。

```
TP3057 串行时序（MSB-first）:
  FSX: ___|‾‾‾|___________________________|‾‾‾|___
  DX:  ...  | b7  b6  b5  b4  b3  b2  b1  b0 | ...
             └── FSX=1 后紧跟的 8 个数据位 ──┘

捕获流程:
  FSX fail_bits → 扫描找到 "1" → DX fail_bits 中紧跟的 8 位 → 组装 PCM 字节
```

### 1.3 代码实现

```c
/* 同时获取 FSX 和 DX 两个通道的 fail memory */
fsx_fail_bits = GET_PATTERN_RESULT(start_idx, fsx_channel, &fsx_depth);
dx_fail_bits  = GET_PATTERN_RESULT(start_idx, dx_channel,  &dx_depth);

for (cycle = offset; cycle < fsx_depth * 8 && byte_idx < num_bytes; cycle++) {
    /* 检测 FSX 上升沿：图形期望 0，实际输出 1 → 失效位=1 */
    if ((fsx_fail_bits[cycle / 8] >> (cycle % 8)) & 1) {
        pcm_byte = 0;
        for (bit = 0; bit < 8; bit++) {
            dx_cycle = cycle + 1 + bit;          /* FSX 之后的第 1~8 周期 */
            dx_level = (dx_fail_bits[dx_cycle / 8] >> (dx_cycle % 8)) & 1;
            pcm_byte |= (unsigned char)(dx_level << (7 - bit));  /* MSB-first */
        }
        pcm_bytes[byte_idx++] = pcm_byte;
    }
}
```

### 1.4 优势

| 对比维度    | 旧方案（legacy）     | 新方案（FSX 监听）                    |
| ----------- | -------------------- | ------------------------------------- |
| 定位方式    | 硬编码固定 offset    | 动态扫描 FSX 失效位                   |
| 图形修改    | 需重新数周期、改代码 | 自动适应                              |
| 鲁棒性      | 依赖绝对周期号       | 依赖 FSX→DX 相对时序（芯片规范保证） |
| offset 语义 | 数据起始位置         | 搜索起始位置（退化）                  |

## 二、`#if` 测试项使能开关

### 2.1 解决的问题

调试单个测试项时，如果所有测试都运行，等待时间长、输出混杂。常规做法是用 `if` 在运行时跳过，但被跳过的代码仍然被编译进二进制文件，占用 ATE 系统的内存和存储空间（ST3020 平台资源有限）。

### 2.2 设计思路

用**编译期条件编译** `#if ... #endif` 替代运行期 `if`。宏置 0 的测试项，整段代码**不参与编译**，生成的二进制文件不包含该段代码，零内存占用。

### 2.3 代码实现

```c
/* ==================== 各测试项独立使能开关 ==================== */
#define TEST_CON_ENABLE       1   /* 连接性测试 */
#define TEST_FUNC_ENABLE      1   /* 功能测试 */
#define TEST_IIL_ENABLE       1   /* IIL 输入低电平漏电流 */
#define TEST_IIH_ENABLE       1   /* IIH 输入高电平漏电流 */
#define TEST_IOZ_ENABLE       1   /* IOZ 三态漏电流 */
#define TEST_ICC0IBB0_ENABLE  1   /* 掉电电流 ICC0/IBB0 */
#define TEST_ICC1IBB1_ENABLE  1   /* 工作电流 ICC1/IBB1 */
#define TEST_GXA_ENABLE       1   /* GXA 发送绝对增益 */
#define TEST_GXR_ENABLE       1   /* GXR 发送增益-频率响应 */
#define TEST_GRA_ENABLE       1   /* GRA 接收绝对增益 */
#define TEST_GRR_ENABLE       1   /* GRR 接收增益-频率响应 */
#define TEST_GRRL_ENABLE      1   /* GRRL 接收增益-电平响应 */
#define TEST_SFDX_ENABLE      1   /* SFDX 发送单频失真 */
#define TEST_SFDR_ENABLE      1   /* SFDR 接收单频失真 */
#define TEST_IMD_ENABLE       1   /* IMD 互调失真 */

/* 使用示例：将不想跑的测试置 0 即可 */
#if TEST_IMD_ENABLE
    {
        /* IMD 所有相关代码：变量声明、波形生成、DVM 采样、DFT、计算 */
        ...
    }
#endif
```

### 2.4 优势

| 对比维度   | `if` 运行时判断 |    `#if` 编译期条件编译    |
| ---------- | :---------------: | :--------------------------: |
| 编译产物   |   包含全部代码   | **只包含开启的测试项** |
| 二进制体积 |       较大       |        **更小**        |
| 内存占用   |  代码+变量均占用  |       **零占用**       |
| 效率       |   每次执行判断   |         零运行时开销         |

### 2.5 典型场景

| 场景     | 配置                     | 说明                         |
| -------- | ------------------------ | ---------------------------- |
| 调试单测 | 只开目标项，其余全部置 0 | 编译快、运行快、结果清晰     |
| 快速验证 | 只开 DC 参数项           | ICC0IBB0 等快速检查供电      |
| 全量生产 | 全部置 1                 | 完整 14 项参数测试           |
| 开发 IMD | 只开 `TEST_IMD_ENABLE` | 跳过其他 13 项，节约等待时间 |

## 三、分箱使能开关

### 3.1 解决的问题

ST3020 的 `BIN(n)` 函数将芯片自动分拣到物理料斗。生产时需要分箱来分离良品/不良品；但**调试时**，我们希望反复测量同一颗芯片来排查问题，一旦触发 `BIN()` 芯片就被踢出，无法继续测试。

### 3.2 设计思路

用宏将 `BIN()` 包装为 `DO_BIN()`，通过 `BIN_ENABLE` 一键切换行为：

- `BIN_ENABLE = 1`（生产模式）：`DO_BIN(n)` → `BIN(n)`，正常分箱
- `BIN_ENABLE = 0`（调试模式）：`DO_BIN(n)` → `((void)0)`，编译为空操作，芯片留在测试位

### 3.3 代码实现

```c
/* ==================== 分箱开关 ==================== */
#define BIN_ENABLE 0               /* 1=生产分箱, 0=调试不分箱 */
#if BIN_ENABLE
  #define DO_BIN(n) BIN(n)
#else
  #define DO_BIN(n) ((void)0)
#endif

/* 使用：代码中只写 DO_BIN，不直接写 BIN */
if (IMD_db > -41.0) {
    DO_BIN(26);        /* BIN_ENABLE=0 时这行不产生任何机器码 */
} else {
    SHOW_RESULT("IMD=>PASS", 1, "", 1, 1);
}
```

### 3.4 优势

| 特性               | 说明                                                      |
| ------------------ | --------------------------------------------------------- |
| **一键切换** | 只改一个宏 `BIN_ENABLE` 的 0/1，不涉及任何测试逻辑代码  |
| **安全**     | 代码中不出现裸 `BIN()`，全部通过 `DO_BIN()`，不会遗漏 |
| **零开销**   | 调试模式下分箱代码完全编译消失，不影响性能                |
| **防误操作** | 生产前将 `BIN_ENABLE` 置 1 即可，不会忘记开启分箱       |

## 四、公共函数封装

### 4.1 解决的问题

14 项测试中有大量重复逻辑：

- SETUP（DPS 上电 + 时序 + FORMAT + 图形 3 上电）每个测试都需要
- A-law PCM 解码在 GXA、GXR、SFDX 中都出现
- 汉宁窗、bin 查找、失真计算在 SFDX、SFDR 中逻辑相同

如果把这些逻辑内联在每个测试项中，修改一处就需要同步修改多处，极易遗漏导致不一致。

### 4.2 设计思路

将通用逻辑抽取为独立函数，每个函数职责单一、接口清晰。所有测试项调用同一份实现。

### 4.3 封装的函数

```c
/* 1. 系统初始化 — 被调用 14 次 */
void SETUP(void) {
    SET_DPS(DPS1, -5.0, V, 100, MA);   /* VBB = -5V */
    SET_DPS(DPS2, 5.0, V, 100, MA);    /* VCC = +5V */
    Delay(10);
    SET_PERIOD(488);                    /* 2.048MHz 主时钟 */
    SET_TIMING(122, 122, 244);          /* 时序边沿 */
    SET_INPUT_LEVEL(2.2, 0.6);          /* 输入电平 */
    SET_OUTPUT_LEVEL(2.4, 0.4);         /* 输出电平 */
    FORMAT(NRZ0, "48,4,47,46,1,2");    /* 通道格式 */
    FORMAT(RZ, "45,44,3");
    RUN_PATTERN(3, 1, 0, 0);           /* 上电，取消 DX 高阻 */
    Delay(10);
}

/* 2. A-law 解码 → 电压 — 被调用 512×n 次 */
double pcm_to_voltage(unsigned char tp3057_byte) {
    alaw_std = tp3057_byte ^ 0x55;     /* TP3057 偶数位反转还原 */
    /* G.711 分段线性解码 → 瞬时电压 */
    ...
}

/* 3. PCM 字节数组 → 真 RMS — 被 GXA/GXR 调用 */
double compute_vrms(unsigned char* pcm_bytes, int count) { ... }

/* 4. 汉宁窗 — 被 SFDX/SFDR/IMD 调用 */
void apply_hanning(double* data, int n) {
    for (i = 0; i < n; i++)
        data[i] *= 0.5 * (1.0 - cos(2π·i/(n-1)));
}

/* 5. 频率 → FFT bin — 被 SFDX/SFDR/IMD 调用 */
int freq_to_bin(double freq, double freq_res, int max_bin) {
    bin = (int)(freq / freq_res + 0.5);   /* 四舍五入取最近 bin */
    return (bin < max_bin) ? bin : -1;     /* 超界返回 -1 */
}

/* 6. 失真 dB 计算 — 被 SFDX/SFDR 调用 */
double calc_distortion_db(double A_fund, double A_harm1, double A_harm2) {
    if (A_fund <= 0.0) return 1;
    return 20.0 * log10(max(A_harm1, A_harm2) / A_fund);
}

/* 7. FSX 动态 PCM 捕获 — 被 GXA/GXR/SFDX 调用 */
int capture_multi_pcm_bytes_fsx(start_idx, dx_ch, fsx_ch, offset, buf, num) { ... }
```

### 4.4 优势

| 方面               | 说明                                                                                                             |
| ------------------ | ---------------------------------------------------------------------------------------------------------------- |
| **修改安全** | 修改一处函数实现，所有调用者同步受益                                                                             |
| **代码简洁** | 每个测试项从 ~100 行缩减到 ~30 行核心逻辑                                                                        |
| **测试独立** | 函数的正确性可单独验证，不依赖任一测试项                                                                         |
| **接口统一** | SFDX/SFDR/IMD 三个失真测试用同一套 `apply_hanning` + `DFT` + `freq_to_bin` + `calc_distortion_db` 流水线 |

### 4.5 新旧代码共存

```c
/* 旧版（基于固定 offset，已弃用但仍保留） */
int capture_multi_pcm_bytes_legacy(start_idx, dx_ch, offset, buf, num) { ... }

/* 新版（基于 FSX 动态检测，当前使用） */
int capture_multi_pcm_bytes_fsx(start_idx, dx_ch, fsx_ch, offset, buf, num) { ... }
```

保留旧版作为历史参照，必要时可回退验证。

## 五、`{ }` 块作用域隔离 14 项测试

### 5.1 解决的问题

14 项测试写在一个 `TP3057()` 函数中，如果所有变量声明在函数顶部，容易出现两类问题：

1. **变量名污染**：GXA 的 `num_byte` 和 SFDX 的 `captured` 如果都叫 `n`，后者覆盖前者的值
2. **残留值误用**：上一测试项设置的变量值被下一个测试项意外读取

### 5.2 设计思路

每个测试项用独立的 `{ }` 块包裹，变量声明在各自块内。C95 的块作用域规则保证块内变量对外不可见。

### 5.3 代码实现

```c
void PASCAL TP3057()
{
    /* ===== Test 01: CON-TEST ===== */
#if TEST_CON_ENABLE
    {
        double v_con;          /* 此变量只在 CON-TEST 块内有效 */
        int i;
        ...
    }                          /* 离开块 → v_con, i 释放 */
#endif

    /* ===== Test 06: GXA ===== */
#if TEST_GXA_ENABLE
    {
        unsigned char pcm_buf_gxa[512];   /* 与下面 GXR 的同名变量互不干扰 */
        int num_byte;                      /* 与 SFDX 的 captured 互不干扰 */
        double gain_db;
        ...
    }
#endif

    /* ===== Test 07: GXR ===== */
#if TEST_GXR_ENABLE
    {
        unsigned char pcm_buf_gxr[512];   /* 独立数组，不会被 GXA 残留覆盖 */
        ...
    }
#endif
    ...
}
```

### 5.4 优势

| 方面                 | 说明                                                   |
| -------------------- | ------------------------------------------------------ |
| **变量名自由** | 每个块内可以自由使用 `i`, `n`, `buf[]` 等常用名  |
| **防止残留**   | 离开作用域后变量生命周期结束，下一个测试项不会读到旧值 |
| **代码清晰**   | `{ }` 边界 = 测试项边界，一目了然                    |
| **无性能开销** | 栈变量分配/释放在编译期确定，运行时无 GC 开销          |

### 5.5 与测试开关的协同

```c
#if TEST_GXA_ENABLE       /* 编译开关：是否包含此测试 */
    {                     /* 作用域开关：变量隔离 */
        int num_byte;
        ...
    }
#endif
```

两层开关各司其职：`#if` 控制**要不要这个测试项**，`{ }` 控制**变量不污染其他测试项**。

## 六、多级失败隔离

### 6.1 解决的问题

测试中可能遇到不同性质的失败：硬件通信失败（图形跑不起来）、算法计算失败（DFT 出错）、芯片性能不达标（指标超限）。如果所有失败统一处理，调试时无法快速定位根因。

### 6.2 设计思路

将失败分为三个等级，每级有不同的处理策略：

```
Level 1: 硬件/通信故障     → 致命，立即 goto 清理代码，中止本测试项
Level 2: 算法/计算故障     → 致命，立即 goto 清理代码，中止本测试项
Level 3: 性能指标超限      → 非致命，记录 BIN，继续后续测试
```

### 6.3 代码实现

以 SFDX 为例完整展示三级隔离：

```c
#if TEST_SFDX_ENABLE
    {
        unsigned char pcm_data[SFDX_FFT_POINTS];
        ...
        SETUP();
        SET_AS(1.2276, V, 1020, HZ);

        RUN_PATTERN(7, 0, 0, 0);

        /* === Level 1: 硬件故障 — PCM 捕获失败 === */
        captured = capture_multi_pcm_bytes_fsx(7, 46, 43, 0, pcm_data, SFDX_FFT_POINTS);
        if (captured != SFDX_FFT_POINTS) {
            SHOW_RESULT("SFDX_CAP_FAIL", (double)captured, "bytes", SFDX_FFT_POINTS, 0);
            DO_BIN(14);
            goto END_SFDX;                       /* ← 直接跳转清理，不继续 */
        }

        /* 解码 + 加窗 ... */

        /* === Level 2: 算法故障 — DFT 失败 === */
        if (!DFT(voltage, SFDX_FFT_POINTS, fft_mag)) {
            DO_BIN(15);
            goto END_SFDX;                       /* ← 直接跳转清理 */
        }

        /* === Level 2: 算法故障 — 谐波超出 FFT 范围 === */
        if (bin_2nd < 0 || bin_3rd < 0) {
            DO_BIN(16);
            goto END_SFDX;                       /* ← 直接跳转清理 */
        }

        /* === Level 3: 性能指标 — 基波丢失 === */
        SFDX_db = calc_distortion_db(...);
        if (SFDX_db >= 0.9) {
            DO_BIN(17);
            /* 不 goto，继续执行（此处已是测试末段） */
        }
        /* === Level 3: 性能指标 — 失真超限 === */
        else if (SFDX_db > -46) {
            DO_BIN(18);
            /* 不 goto，继续执行 */
        }

    END_SFDX:                                    /* 统一清理入口 */
        DPS_OFF(DPS1);
        DPS_OFF(DPS2);
    }
#endif
```

### 6.4 各测试项的 BIN 分配

| 测试项 | Level 1 (硬件) |            Level 2 (算法)            |  Level 3 (超限)  |
| ------ | :-------------: | :-----------------------------------: | :---------------: |
| SFDX   |  14 (捕获失败)  | 15 (DFT), 16 (bin越界), 17 (基波丢失) |   18 (失真超限)   |
| SFDR   | 31 (图形10失败) | 19 (DFT), 20 (bin越界), 21 (基波丢失) |   22 (失真超限)   |
| IMD    | 32 (图形13失败) | 23 (DFT), 24 (bin越界), 25 (基波丢失) |   26 (IMD超限)   |
| GXA    |  27 (捕获失败)  |                  —                  |   28 (增益超限)   |
| GXR    |  29 (捕获失败)  |                  —                  |   30 (频响超限)   |
| GRA    |       —       |                  —                  |   11 (增益超限)   |
| GRR    |       —       |                  —                  |   12 (频响超限)   |
| GRRL   |       —       |                  —                  | 13 (电平响应超限) |

### 6.5 CON-TEST 前置拦截

```c
#if TEST_CON_ENABLE
    {
        ...
        if (!PMU_MEASURE("1,2,3,4,5,41,42,43,44,45,46,47,48", 20, "CON_", V, -0.1, -1.9)) {
            SHOW_RESULT("CON-TEST=>FAIL", 1, "", 1, 1);
            DO_BIN(1);
            CLEAR_RELAY("3,4,5,6");
            DPS_OFF(DPS1);
            DPS_OFF(DPS2);
            return;    /* ← 直接退出整个 TP3057(), 跳过后续 13 项测试 */
        }
        ...
    }
#endif
```

CON-TEST 是唯一使用 `return` 而非 `goto END_XXX` 的测试——因为它不探测具体参数，而是判断芯片是否物理完好。损坏的芯片不值得继续测试。

### 6.6 优势

| 方面                   | 说明                                                                                  |
| ---------------------- | ------------------------------------------------------------------------------------- |
| **故障定位精确** | 32 个 BIN，每个对应一条具体的失败路径，无需重新测量即可知道"在哪一步失败了"           |
| **资源节约**     | Level 1/2 触发后立即终止本项 → 跳过无用计算 → 节约测试时间                          |
| **测试独立**     | Level 3 不终止 → 后续测试项不受影响 → 一颗芯片可拿到全部参数报告                    |
| **安全保障**     | `goto END_XXX` 跳转到统一的 `CLEAR_RELAY + DPS_OFF`，不会残留继电器状态或带电引脚 |

## 七、特色功能总览

| # | 特色功能                       | 解决什么问题                 | 关键技术                                             |
| :-: | ------------------------------ | ---------------------------- | ---------------------------------------------------- |
| 1 | **FSX 监听定位 PCM**     | 硬编码 offset 随图形修改失效 | Fail Memory 双通道扫描 + 帧同步检测                  |
| 2 | **`#if` 测试使能开关** | 调试效率低、无用代码占用资源 | 编译期条件编译，关闭的代码零体积                     |
| 3 | **分箱使能开关**         | 调试/生产环境需求冲突        | 宏包装 `BIN()`，一键切换行为                       |
| 4 | **公共函数封装**         | 重复代码散落、修改易遗漏     | 7 个单一职责函数，SFDX/SFDR/IMD 复用同一流水线       |
| 5 | **`{ }` 块作用域隔离** | 14 项测试变量污染风险        | C95 块作用域，每个测试项变量独立                     |
| 6 | **多级失败隔离**         | 不同性质失败混在一起难以定位 | 3 级分级处理：硬件→goto、算法→goto、指标→记录继续 |
